# Governance

Governance here means practical operating controls for responsible experimentation.

## Operating controls

- clear ownership for ingestion, retrieval, orchestration, and review
- versioned prompts, policies, and evaluation artefacts
- release notes for behaviour changes affecting risk profile
- incident review workflow with corrective actions

## Decision rights

- humans approve high-impact outputs
- system automation is bounded by policy thresholds
- unresolved uncertainty triggers escalation by default

## Documentation expectations

Before claiming an improvement, capture:

1. what changed
2. why it changed
3. how it was evaluated
4. known limitations and follow-up tasks

Use [`templates/incident-review.md`](../templates/incident-review.md) for post-incident analysis.
